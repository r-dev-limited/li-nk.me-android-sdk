LinkMe Android SDK

Deep linking, deferred deep linking, and attribution for Android apps. Get users to the right content — even on first install.

Setup Guide Maven Central

1 Install

// app/build.gradle.kts
dependencies {
    implementation("me.li-nk:linkmekit:0.2.13")
}

2 Initialize

LinkMe.shared.configure(
    context = applicationContext,
    config = LinkMe.Config(
        appId = BuildConfig.LINKME_APP_ID,
        appKey = BuildConfig.LINKME_APP_KEY,
        debug = BuildConfig.DEBUG,
    ),
)

3 Handle Links

// Cold-start link
LinkMe.shared.getInitialLink { payload ->
    payload?.let(::routeUser)
}

// Deferred deep link (first install)
LinkMe.shared.claimDeferredIfAvailable(this) { deferred ->
    deferred?.let(::routeUser)
}

// Live links while app is open
LinkMe.shared.addListener { payload ->
    routeUser(payload)
}

Features

App Links

Verified HTTPS deep links via Android App Links. LinkMe hosts assetlinks.json automatically.

Deferred Deep Linking

Route users to the right content on first install via Play Install Referrer or fingerprint fallback.

Analytics

Track app events and attribute installs to campaigns with track() and UTM parameters.

Zero Config

Install Referrer and Ad ID libraries are bundled. No extra dependencies needed.
